# ☁️ Amazon EC2 (Elastic Compute Cloud)

## 📌 Introduction

Amazon EC2 (Elastic Compute Cloud) is a cloud computing service provided by AWS that allows users to create and manage virtual servers in the cloud.


---

## 🧠 What is Amazon EC2?

Amazon EC2 enables developers and organizations to run applications on **virtual machines**, called **instances**, instead of using physical hardware.

These instances can be:

- Launched anytime  
- Configured as needed  
- Resized based on requirements  
- Terminated when no longer needed  


---

## 🚀 Why EC2 is Important

EC2 is one of the most important AWS services because:

- It provides **on-demand computing power**  
- It forms the backbone of many cloud applications  
- It allows scalable and flexible infrastructure  


---

## ⚙️ Amazon EC2 Overview

Amazon EC2 allows users to:

- Launch virtual machines within minutes  
- Choose operating systems (Linux or Windows)  
- Configure CPU, RAM, and storage  
- Scale infrastructure automatically  
- Pay only for the computing capacity used  


---

## 🏗️ EC2 Architecture

EC2 works within the AWS infrastructure:

- Region → Geographic location  
- Availability Zone → Data center inside region  
- Instance → Virtual machine  

Example:

Region (Mumbai)
   → AZ (ap-south-1a)
      → EC2 Instance


---

## ⚡ How Amazon EC2 Works

1. Choose an Amazon Machine Image (AMI)  
   → Pre-configured OS (Linux/Windows)

2. Select Instance Type  
   → Defines CPU, RAM, performance  

3. Configure Instance  
   → Network, storage, security  

4. Launch Instance  
   → Virtual server starts running  

5. Connect to Instance  
   → Using SSH (Linux) or RDP (Windows)  

6. Deploy Applications  
   → Install software, host websites, run apps

---

## 🔄 EC2 Workflow

The EC2 workflow includes the following steps:

1. Selecting an Amazon Machine Image (AMI)  
2. Choosing an Instance Type  
3. Configuring networking and security  
4. Launching the virtual server  
5. Connecting via SSH (Linux) or Remote Desktop (Windows)  

Users can start, stop, or terminate instances anytime based on their needs.
